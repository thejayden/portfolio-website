# Jia Dian's Portfolio

Personal portfolio site built with [Jekyll](https://jekyllrb.com/), based on the
[Hyperspace](https://html5up.net/hyperspace) theme by HTML5 UP, with
[Tailwind CSS](https://tailwindcss.com/) utilities layered on top.

Hosted on GitHub Pages at: **https://thejayden.github.io/portfolio-website/**

---

## Project Structure

```
.
├── _data/
│   └── projects.yml          # All project entries — edit here to add/update projects
│
├── _includes/
│   ├── head.html             # Shared <head> (meta tags, CSS links)
│   ├── sidebar.html          # Sidebar navigation with avatar
│   └── footer.html           # Footer with dynamic copyright year
│
├── _layouts/
│   └── default.html          # Base page shell (HTML/body wrapper + scripts)
│
├── _scripts/                 # One-off helper scripts (not included in the build)
│
├── assets/
│   ├── css/
│   │   ├── main.css          # Compiled Hyperspace theme CSS (do not edit directly)
│   │   ├── custom.scss       # Custom styles — edit this for style changes
│   │   ├── tailwind.css      # Tailwind input file (utilities layer only)
│   │   └── tailwind.min.css  # Generated by CI / npm run build:css
│   ├── js/
│   │   ├── typewriter.js     # Typewriter animation for the intro heading
│   │   └── ...               # Hyperspace vendor scripts
│   └── sass/                 # Hyperspace theme SCSS source
│
├── images/                   # All image assets
│
├── .github/workflows/
│   └── deploy.yml            # GitHub Actions: builds Tailwind + Jekyll, deploys to Pages
│
├── index.html                # Main page (uses Jekyll layout + loops over _data/projects.yml)
├── equinix.html              # Equinix internship detail page
├── _config.yml               # Jekyll configuration
├── Gemfile                   # Ruby dependencies (Jekyll, plugins)
└── package.json              # Node dependencies (Tailwind CSS)
```

## Adding or Editing a Project

All projects are driven from `_data/projects.yml`. No HTML editing needed.

Each entry follows this shape:

```yaml
- title: "My Project Title"
  image: "images/my-image.jpg"
  imagePosition: "center center"   # CSS background-position value
  description: >
    A description of the project.
  links:
    - url: "https://github.com/..."
      label: "GitHub"
    - url: "https://myproject.com"
      label: "Website"
  tags:
    - "React"
    - "TypeScript"
```

## Getting Started Locally

### Prerequisites (one-time setup)

You need **Ruby 3.x**, **Bundler**, and **Node 18+**.

The system Ruby on macOS is too old. Use `rbenv` to install a modern version:

```bash
# Install rbenv
brew install rbenv ruby-build
echo 'eval "$(rbenv init - zsh)"' >> ~/.zshrc
source ~/.zshrc

# Install Ruby 3.3 and pin it to this project
rbenv install 3.3.0
cd /path/to/portfolio-website
rbenv local 3.3.0
```

### Install dependencies

```bash
gem install bundler
bundle install    # installs Jekyll + plugins
npm install       # installs Tailwind CSS
```

### Run locally

Open two terminals in the project directory:

**Terminal 1 — Tailwind watcher:**
```bash
npm run watch:css
```

**Terminal 2 — Jekyll dev server:**
```bash
bundle exec jekyll serve --livereload
```

Visit **http://localhost:4000/portfolio-website/**

> Pass `--baseurl ""` to serve at `http://localhost:4000/` instead:
> `bundle exec jekyll serve --livereload --baseurl ""`

## Deploying to GitHub Pages

Deployment is fully automated via GitHub Actions (`.github/workflows/deploy.yml`).

Every push to `main` or `master`:
1. Installs Node dependencies and builds Tailwind CSS
2. Installs Ruby dependencies and builds the Jekyll site
3. Deploys the `_site/` output to GitHub Pages

### One-time GitHub setup

In your repository go to **Settings → Pages → Source** and select
**"GitHub Actions"** (not "Deploy from a branch").

That's it — the next push will trigger a deployment automatically.
